# v0.2 变更统计

## 一、新增功能

### 1. FastAPI 接口
- 新增文件: `src/api/__init__.py`, `src/api/app.py`, `src/api/server.py`
- RESTful API 端点:
  - `GET /health` — 健康检查
  - `POST /tasks` — 执行任务（JSON 请求）
  - `POST /tasks/upload` — 执行任务（文件上传）
  - `GET /templates` — 列出工作流模板
  - `GET /templates/{name}` — 获取模板详情
  - `GET /config` — 获取配置信息
- 启动方式: `python app.py --api`
- API 文档: http://127.0.0.1:8000/docs

### 2. YAML 工作流模板
- 新增文件: `src/workflow/templates.py`
- 默认模板目录: `workflows/`
- 内置模板:
  - `summarize.yaml` — 文档总结工作流
  - `extract_todos.yaml` — 待办提取工作流
  - `risk_analysis.yaml` — 风险分析工作流
  - `meeting_notes.yaml` — 会议纪要工作流
- API:
  - `list_templates()` — 列出所有模板
  - `load_workflow_template(name)` — 加载模板
  - `create_template(name, description, steps)` — 创建模板
  - `delete_template(name)` — 删除模板

### 3. 用户偏好记忆
- 新增文件: `src/preferences/__init__.py`, `src/preferences/manager.py`
- 存储位置: `data/preferences/user_preferences.json`
- 可配置项:
  - `default_mode` — 默认处理模式
  - `default_output_format` — 默认输出格式
  - `include_metadata` — 是否包含元数据
  - `show_token_count` — 是否显示 token 计数
  - `auto_preview` — 是否自动预览
  - `favorite_templates` — 收藏的模板列表
  - `recent_templates` — 最近使用的模板
- API:
  - `load()` — 加载偏好
  - `save(preferences)` — 保存偏好
  - `update(**kwargs)` — 更新特定字段
  - `add_recent_template(name)` — 添加最近使用模板
  - `toggle_favorite_template(name)` — 切换收藏状态
  - `reset()` — 重置为默认值

### 4. 多格式输出支持
- 扩展文件: `src/workflow/postprocess.py`
- 支持格式:
  - `markdown` — 标准 Markdown（默认）
  - `plain` — 纯文本（去除格式标记）
  - `json` — 结构化 JSON（含 sections 和 metadata）
  - `html` — 基础 HTML 格式
- API: `format_output(raw_output, fmt, metadata)`

## 二、配置变更

### config.yaml 新增配置项
```yaml
# 输出格式
output:
  format: markdown  # markdown, plain, json, html

# 工作流模板
workflow:
  templates_dir: workflows

# 用户偏好
preferences:
  dir: data/preferences

# FastAPI 服务
api:
  host: 127.0.0.1
  port: 8000
```

### AppConfig 新增字段
- `default_output_format: str` — 默认输出格式
- `workflows_dir: str` — 工作流模板目录
- `preferences_dir: str` — 用户偏好目录
- `api_host: str` — API 服务地址
- `api_port: int` — API 服务端口

## 七、技术亮点

1. **模块化设计**: FastAPI 接口、工作流模板、用户偏好各自独立模块
2. **向后兼容**: v0.1 所有功能保持不变，新功能为增量添加
3. **完整测试**: 每个新模块都有对应测试，确保代码质量
4. **灵活配置**: 通过 config.yaml 和环境变量均可配置
5. **RESTful API**: 标准化接口，便于前端/桌面端接入

## 八、使用示例

### 启动 FastAPI 服务
```bash
python app.py --api
```

### 使用工作流模板执行任务
```bash
curl -X POST http://127.0.0.1:8000/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "query": "分析会议记录",
    "workflow_template": "meeting_notes",
    "files": ["data/meeting.md"]
  }'
```

### 列出可用模板
```bash
curl http://127.0.0.1:8000/templates
```

### 使用不同输出格式
```python
from src.workflow.postprocess import format_output

# Markdown 格式
md_output = format_output(raw_text, fmt="markdown")

# JSON 格式
json_output = format_output(raw_text, fmt="json", metadata={"model": "deepseek-v4-flash"})

# HTML 格式
html_output = format_output(raw_text, fmt="html")
```
