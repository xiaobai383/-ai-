# v0.1 → v0.2 升级方案

## 一、v0.2 目标

根据最终方案.md第十二节版本路线：

> v0.2 | 个人偏好与工作流模板 | YAML 工作流、默认输出格式、用户偏好记忆、FastAPI

---

## 二、新增/优化功能清单

### 2.1 FastAPI REST API（新增）

**优先级**: P2  
**目标**: 提供 HTTP API 接口，方便后续接入桌面端或 Web 前端

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /api/run` | POST | 执行任务（接收 query、files、mode） |
| `GET /api/runs` | GET | 获取历史 RunLog 列表 |
| `GET /api/runs/{run_id}` | GET | 获取单次执行详情 |
| `GET /api/config` | GET | 获取当前配置 |
| `GET /api/health` | GET | 健康检查 |

**新增文件**:
- `src/api/__init__.py`
- `src/api/server.py` — FastAPI app 定义
- `src/api/schemas.py` — Pydantic 请求/响应模型

**依赖变更**:
- `requirements.txt` 添加 `fastapi>=0.110.0` 和 `uvicorn>=0.27.0`

---

### 2.2 YAML 工作流模板（新增）

**优先级**: P2  
**目标**: 用户可预定义常用工作流模板，一键复用

**模板格式示例** (`workflows/summary.yaml`):
```yaml
name: 文档摘要
description: 提取文档要点、待办、风险点
mode: privacy_enhanced
prompt_template: |
  请分析以下文档，提取：
  1. 核心要点（不超过5条）
  2. 待办事项
  3. 潜在风险
output_format: markdown
save_to: output/
```

**新增文件**:
- `src/workflow/__init__.py`（已有，需扩展）
- `src/workflow/templates.py` — 模板加载与管理
- `workflows/` 目录 — 存放用户自定义模板

**功能点**:
- 从 `workflows/` 目录加载 `.yaml` 模板
- UI 模板选择器（Gradio 下拉框）
- API 支持 `template` 参数
- 模板中可指定 `mode`、`prompt_template`、`output_format`、`save_to`

---

### 2.3 用户偏好记忆（新增）

**优先级**: P2  
**目标**: 记住用户的常用设置，减少重复选择

**偏好项**:
| 偏好 | 说明 | 默认值 |
|------|------|--------|
| `default_mode` | 默认运行模式 | `privacy_enhanced` |
| `default_output_format` | 默认输出格式 | `markdown` |
| `auto_confirm` | 是否自动确认上传 | `false` |
| `default_model` | 默认模型 | 从 config.yaml 读取 |
| `theme` | UI 主题 | `light` |

**存储方式**: `data/preferences.json`

**新增文件**:
- `src/config.py` 扩展 — 添加 `UserPreferences` dataclass
- `src/ui/gradio_app.py` 扩展 — 偏好设置面板

---

### 2.4 默认输出格式（优化）

**优先级**: P2  
**目标**: 支持多种输出格式，用户可选择

**支持格式**:
- `markdown`（默认）— 保存为 `.md`
- `plain` — 保存为 `.txt`
- `json` — 结构化 JSON 输出

**修改文件**:
- `src/workflow/postprocess.py` — 扩展 `format_output()` 支持多格式
- `src/ui/gradio_app.py` — 添加格式选择控件

---

### 2.5 PDF/DOCX 文件支持（优化）

**优先级**: P1（加分项）  
**目标**: 扩展文件解析能力

**新增依赖**:
- `PyPDF2>=3.0.0` — PDF 解析
- `python-docx>=1.0.0` — DOCX 解析

**修改文件**:
- `src/tools/file_ops.py` — `parse_file()` 添加 `.pdf` / `.docx` 分支
- `src/ui/gradio_app.py` — 文件选择器添加 `.pdf` / `.docx` 类型

---

### 2.6 RunLog 持久化（优化）

**优先级**: P1  
**目标**: RunLog 保存到 `data/logs/` 目录，支持历史查询

**修改文件**:
- `src/observability/run_log.py` — 添加 `save_to_disk()` 方法
- `src/agent/app.py` — 执行完成后自动保存 RunLog

---

### 2.7 流式反馈增强（优化）

**优先级**: P1  
**目标**: UI 更细粒度的执行进度展示

**改进点**:
- 使用 Gradio 的 `gr.Progress` 替代纯文本进度
- 每个步骤实时更新状态图标
- 添加预估剩余时间
