# v0.3 变更统计

## 一、新增功能

### 1. 文件夹监听
- 新增文件: `src/monitor/__init__.py`, `src/monitor/watcher.py`
- 基于 polling 的文件系统变更检测
- 支持两种触发模式:
  - `instant` — 文件新增/修改即触发
  - `batch` — 时间窗口内累积，窗口到期后批量触发
- 文件模式过滤（`*.txt`, `*.md`）
- API 端点:
  - `POST /watch/start` — 启动监听
  - `POST /watch/stop` — 停止监听
  - `GET /watch/status` — 查询状态

### 2. 定时任务调度
- 新增文件: `src/scheduler/__init__.py`, `src/scheduler/engine.py`, `src/scheduler/jobs.py`
- 基于 APScheduler 的 BackgroundScheduler
- 预设模板: 每天9点、每天18点、每小时、工作日早上、每周一
- 支持自定义 cron/interval 触发器
- 任务 JSON 持久化（`data/scheduled_jobs/`）
- 任务暂停/恢复
- API 端点:
  - `GET /scheduler/jobs` — 列出任务
  - `POST /scheduler/jobs` — 创建任务（从预设）
  - `DELETE /scheduler/jobs/{job_id}` — 删除任务
  - `POST /scheduler/jobs/{job_id}/pause` — 暂停
  - `POST /scheduler/jobs/{job_id}/resume` — 恢复

### 3. 批量处理
- 新增文件: `src/batch/__init__.py`, `src/batch/runner.py`
- `BatchRunner` 串行执行多个文件的 `run_task`
- `BatchReport` 聚合统计: 总数/成功/失败/token/费用/耗时
- 支持进度回调
- API 端点: `POST /batch`

### 4. 桌面通知
- 新增文件: `src/notify/__init__.py`, `src/notify/dispatch.py`
- 双引擎: `plyer` 桌面通知 + JSONL 日志兜底
- 模式: `auto`（优先 plyer）| `plyer` | `log`
- 任务完成/失败自动触发（集成到 `run_task` 末尾）
- API 端点: `GET /notifications` — 查询通知历史

## 二、配置变更

### config.yaml 新增配置项
```yaml
watch:
  enabled: false
  dirs: [data/watch/]
  patterns: ["*.txt", "*.md"]
  trigger_mode: instant
  batch_window_seconds: 60
  workflow_template: summarize
  mode: privacy_enhanced

scheduler:
  enabled: false
  jobs_dir: data/scheduled_jobs

notifications:
  enabled: true
  engine: auto
  log_file: data/logs/notifications.jsonl
```

### AppConfig 新增字段
- `watch_enabled`, `watch_dirs`, `watch_patterns`, `watch_trigger_mode`, `watch_batch_window_seconds`, `watch_workflow_template`, `watch_mode`
- `scheduler_enabled`, `scheduler_jobs_dir`
- `notifications_enabled`, `notifications_engine`, `notifications_log_file`

## 七、技术亮点

1. **零侵入集成** — 所有 v0.3 模块均可独立启用/禁用，不影响 v0.1/v0.2 功能
2. **配置驱动** — 监听/调度/通知均通过 config.yaml 控制
3. **优雅降级** — 通知 plyer 失败自动降级到 JSONL 日志
4. **持久化设计** — 定时任务保存为独立 JSON 文件，重启不丢失
5. **标签页 UI** — 单任务/自动/批量/通知四大标签页，功能分组清晰
