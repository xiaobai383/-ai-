# v1.0 变更统计

## 一、新增功能

### 1. 向量知识库（`src/knowledge/`）
- 新增模块: `src/knowledge/store.py` — ChromaDB 持久化存储，管理 RunLog / 输出 / 偏好三个 collection
- 新增: `src/knowledge/embedder.py` — Ollama nomic-embed-text 嵌入封装，含可用性检测与结果缓存
- 新增: `src/knowledge/indexer.py` — 批量索引器，遍历 `data/logs/` 和 `output/`，基于文件 cache key 增量更新
- 新增: `src/knowledge/search.py` — `Searcher` 语义检索接口，返回带相似度分数的 `SearchResult` 列表
- Config: `knowledge` 配置节（chroma 路径、embedding 模型、base_url、默认 top_k）
- API: `GET /knowledge/search` — 语义检索端点
- 依赖: `chromadb>=0.5.0`

### 2. 本地模型兜底（`src/fallback/`）
- 新增: `src/fallback/provider.py` — `FallbackChatModel` 包装类，继承 LangChain `BaseChatModel`
- 策略: 先调云端 API → 失败（超时/5xx/网络错误）自动切 Ollama 本地模型
- RunLog 增强: 新增 `fallback: bool` 字段，JSONL 序列化/反序列化完整支持
- Config: `fallback` 配置节（enabled、ollama_base_url、模型名、超时）
- 集成: `src/ui/gradio_app.py` 和 `src/api/app.py` 自动使用 FallbackChatModel

### 3. 执行回放（`src/replay/`）
- 新增: `src/replay/loader.py` — `RunLogLoader` 扫描 `data/logs/`，按模式/状态/关键词筛选搜索
- 新增: `RunLogSummary` 轻量摘要数据类，支持分页
- UI: 工作台标签页内嵌回放时间线，支持搜索 + 模式/状态筛选
- API: `GET /replay/runs`（列表） + `GET /replay/runs/{run_id}`（详情）

### 4. 交互仪表盘（`src/dashboard/`）
- 新增: `src/dashboard/aggregator.py` — `DashboardAggregator` 从 JSONL 聚合全量统计
- 新增: `src/dashboard/charts.py` — Plotly 图表工厂（费用趋势折线、模式/模型饼图、Token 堆叠柱状图、空图占位）
- UI: 洞察标签页含 KPI 卡片 + 4 个交互图表 + 最近任务列表 + 时间范围筛选
- API: `GET /dashboard/stats` — 聚合统计端点
- 依赖: `plotly>=5.18.0`

### 5. RunLog 持久化增强
- 新增: `RunLog.save_to_disk(path)` — 便捷保存方法，自动创建目录
- 增强: `run_task` 执行完成后自动调用 `save_to_disk("data/logs")` 持久化 RunLog

### 6. UI 重组：7→4 标签页
| 旧标签 | → | 新标签 |
|--------|---|--------|
| 单任务处理 | → | 🛠️ **工作台**（单任务 + 执行回放） |
| 自动处理 / 批量处理 | → | （暂未集成到 v1.0 UI，仍可通过 API 使用） |
| 知识检索 | → | 📚 **知识库**（语义检索 + 索引管理） |
| 仪表盘 / 通知历史 | → | 📊 **洞察**（仪表盘 + 通知） |
| 设置 | → | ⚙️ **设置**（配置摘要 + 模板 + 偏好） |

### 7. API 版本升级
- 版本号: `0.3.0` → `1.0.0`
- 新增 5 个端点: `GET /replay/runs`、`GET /replay/runs/{id}`、`GET /knowledge/search`、`GET /dashboard/stats`

## 二、配置变更

### config.yaml 新增配置项
```yaml
# v1.0 — knowledge retrieval
knowledge:
  chroma_dir: data/chroma
  embed_model: nomic-embed-text
  embed_base_url: http://localhost:11434
  default_top_k: 5

# v1.0 — local model fallback
fallback:
  enabled: true
  ollama_base_url: http://localhost:11434/v1
  ollama_model: qwen2.5:1.5b
  timeout_seconds: 10
```

## 六、技术亮点

1. **ChromaDB 零配置** — pip install 即用，vector DB 无外部服务依赖
2. **Ollama 统一栈** — embedding 和 LLM fallback 共用同一个 Ollama 实例
3. **透明降级** — FallbackChatModel 对调用方完全透明，UI 和 API 均无需改动调用逻辑
4. **增量索引** — 基于文件 cache key（path + mtime + size）跳过已索引文件，避免重复 embedding
5. **Plotly 交互图表** — 可缩放、悬停查看数值的仪表盘，Gradio 原生支持
6. **249 测试全绿** — 所有模块均有完整测试覆盖，零回归
